  - 桌面模式：`python run.py --mode desktop` 在原生窗口中运行
  - 打包后：双击 `dist/TimerFocus.exe`（17MB）自动启动桌面窗口

## 四、关键设计决策

### 1. 双模式启动

#### 为什么这样设计

保持开发体验不变的前提下，新增桌面模式的入口。通过 `--mode` 参数区分：

- `dev`（默认）：原有 Flask dev server，带 debug 热重载，前端仍由 Vite 提供
- `desktop`（PyInstaller 打包后默认）：Flask 在后台线程运行，PyWebView 创建原生桌面窗口

打包后 `sys.frozen` 为 True，自动切换默认模式为 `desktop`。

#### 影响

开发流程零变化。打包后的 .exe 双击直接出桌面窗口。

### 2. 前端静态文件托管

#### 为什么这样设计

打包后不再有 Vite 开发服务器，Flask 需要自己提供前端页面。`register_static_routes()` 函数在检测到 `frontend/dist/` 目录存在时自动注册：

- `/` → index.html
- `/assets/*` → 静态资源
- 其他路径 → 回退到 index.html（支持 Vue Router history 模式）

打包时通过 `--add-data "frontend/dist;frontend/dist"` 将前端产物嵌入 .exe，运行时通过 `sys._MEIPASS` 找到解压路径。

#### 影响

打包后浏览器不再需要，PyWebView 直接加载 Flask 提供的页面。

### 3. 体积优化

#### 第一次打包：324MB

Anaconda 环境自动拉入了 NumPy、Matplotlib、PyQt5、IPython、Sphinx 等大量无关库。

#### 优化后：17MB

通过 `--exclude-module` 排除科学计算和开发工具库，Flask + SQLAlchemy + Jinja2 + PyWebView 的核心依赖仅 ~17MB。

## 五、问题与解决方案

### 问题1：首次打包 324MB

#### 现象

PyInstaller 打包输出 324MB 的 .exe。

#### 根因

Anaconda 环境包含大量科学计算库（NumPy、SciPy、Matplotlib、PyQt5），PyInstaller 自动分析依赖时全部打包。

#### 解决方案

使用 `--exclude-module` 明确排除 TimerFocus 不依赖的包。核心依赖只有 Flask、SQLAlchemy、Jinja2、Werkzeug、PyWebView。

#### 最终结果

17MB，可接受。

### 问题2：打包后前端路径找不到

#### 现象

PyInstaller 打包后运行 .exe，`frontend/dist/` 路径错误。

#### 根因

PyInstaller 将文件解压到临时目录（`sys._MEIPASS`），硬编码的项目相对路径失效。

#### 解决方案

`get_frontend_dir()` 函数通过 `sys.frozen` 判断运行环境，动态返回正确的目录路径。

#### 最终结果

开发模式和打包模式均正常找到前端文件。

## 八、开发复盘

### 本次最大收获

PyInstaller + PyWebView 组合是 Python Web 应用转桌面应用的高效方案。核心改动只有 3 个文件，不需要重写任何业务逻辑。Anaconda 环境不适合直接打包，排除无关库可将体积从 300MB+ 降到 20MB 以内。

### 本次踩过的坑

PyInstaller 默认分析所有已安装包，Anaconda 的"全家桶"特性导致体积爆炸。正确的做法是用 `--exclude-module` 排除不需要的，或者用干净的 venv 环境。

### 如果重新设计

建议在项目初期就建立一个独立的 `venv` 环境，只安装 `requirements.txt` 中的依赖，打包体积会更小（预计 10MB 以内）。

## 九、打包问题排查记录

### 问题1：`ModuleNotFoundError: No module named 'webview'`

- **根因**：(1) PyInstaller 使用 base 环境（无 pywebview）；(2) `import webview` 在函数内部，静态分析不到
- **修复**：(1) 改用 FTimer conda 环境；(2) 将 import 移到文件顶部；(3) 使用 `--collect-all webview`

### 问题2：`UnicodeEncodeError: 'gbk' codec can't encode '❌'`

- **根因**：Windows 控制台默认 GBK 编码，emoji 字符无法输出
- **修复**：(1) 添加 `sys.stdout.reconfigure(encoding='utf-8', errors='replace')`；(2) 所有 print 改用纯 ASCII

### 问题3：`DLL load failed while importing _ssl`

- **根因**：conda 环境的 OpenSSL DLL（libssl-3-x64.dll, libcrypto-3-x64.dll）未被 PyInstaller 发现
- **修复**：使用 `--add-binary` 显式添加两个 DLL 文件
