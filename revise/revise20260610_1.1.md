## 一、概述

### 做了什么

基于 **UI-UX-Pro-Max** 设计智能工具包，对 TimerFocus 前端进行了全面的视觉重设计。核心变化：

- 引入 **Exaggerated Minimalism** 设计风格（粗体极简、高对比度、负空间、超大字号）
- 替换全新色彩系统：浅色模式使用生产力工具 Teal 调色板，深色模式使用 Timer & Pomodoro 红黑调色板（Focus Red + Break Green）
- 全局字体从系统默认替换为 **Plus Jakarta Sans**（现代友好、专为生产力工具设计）
- 统一设计变量体系（CSS Variables）：圆角、阴影、间距、字号、过渡动画
- 仅修改前端样式/CSS，**后端代码一行未动**

### 为什么做

原有前端界面完全照搬需求文档（req.md）中的基础配色方案，视觉上缺乏设计感：
- 色彩过于普通（Element Plus 默认蓝 `#409eff`，灰底白卡）
- 字体为系统默认（`-apple-system`），没有品牌识别度
- 卡片/组件间距不统一，缺乏层次感
- 深色模式颜色对比度不足

引入专业设计系统后，界面在视觉质量和用户体验上有质的飞跃。

### 涉及模块

- `frontend/src/styles/` — 全局主题变量 + 全局样式
- `frontend/src/views/` — 5个页面样式（Home/History/Stats/Tags/Settings）
- `frontend/src/components/` — 18个组件样式（common/timer/records/stats/settings）
- `frontend/src/utils/chartConfig.js` — ECharts 图表配色
- `frontend/index.html` — 字体预加载 + meta 优化

### 最终效果

- 构建成功（`npm run build` 通过）
- 浅色模式：清新 Teal 绿，温暖卡白，柔和阴影
- 深色模式：专注红黑，品牌识别度高，OLED 友好
- 所有过渡动画流畅（Bezier 缓动）
- 完全向后兼容，不影响任何功能逻辑

## 四、关键设计决策

### 1. 采用 CSS Variables 统一设计变量

#### 为什么这样设计

在 `theme.css` 中集中定义所有设计变量（颜色、圆角、阴影、间距、字号、过渡），然后在所有组件中通过 `var(--xxx)` 引用。这样做的好处：

- **一致性**：所有组件自动继承同一套设计语言，杜绝硬编码颜色
- **可维护性**：修改一个变量值即可全局生效，如调整主色调只需改 `--primary-color`
- **主题切换**：深色/浅色模式只需在 `[data-theme="dark"]` 下覆盖变量即可

#### 替代方案

- 使用 SCSS mixin/variable — 需要在每个组件中 `@import`，构建后变量不会出现在运行时，无法支持动态主题切换
- 使用 Element Plus 内置主题 — 扩展性有限，不支持自定义 CSS 属性
- CSS-in-JS（如 styled-components）— 项目使用 `<style scoped>` 模式，引入额外依赖成本过高

#### 影响

- Vue 组件 `<style scoped>` 可直接使用 `var(--primary-color)` 等变量
- Element Plus 深色模式覆盖在 `global.scss` 中通过 `[data-theme="dark"]` 选择器统一管理
- 图表配色在 `chartConfig.js` 中读取模式参数后返回对应颜色数组

### 2. 选择 Exaggerated Minimalism 风格

#### 为什么这样设计

UI-UX-Pro-Max 搜索将 Timer & Pomodoro 应用匹配到 Exaggerated Minimalism 风格，原因是：

- 计时器应用的核心交互非常单一（数字 + 按钮），极简风格避免视觉干扰
- 超大字号（`font-size: 72px, font-weight: 300`）让计时数字成为绝对视觉焦点
- 高对比度和充足负空间让用户一目了然
- 轻量动画（Bezier 缓动、脉动、发光）提供微妙反馈而不喧宾夺主

#### 替代方案

- **Dark Mode (OLED)** — 仅支持深色模式，不适合需要浅色模式的办公场景
- **Neumorphism** — 软阴影风格虽然现代，但对视障用户不友好（对比度低），WCAG 合规性差
- **Bento Box Grid** — 适合仪表盘和数据展示，但计时器核心是数字而非信息密度

#### 影响

- 计时数字从 `font-weight: 200` 改为 `300`，从系统 mono 字体改为 Plus Jakarta Sans
- 卡片统一使用 `border-radius: 12-16px`（圆角变大），`box-shadow` 增加层级感
- 顶部渐变条（`::before` 伪元素）为计时面板增加品牌识别

### 3. Plus Jakarta Sans 字体选择

#### 为什么这样设计

UI-UX-Pro-Max 对 Productivity Tool 类型的字体推荐结果就是 Plus Jakarta Sans，其特点：

- 现代几何感 + 人文温暖感平衡
- 数字 tabular-nums 特性适合计时器
- Google Fonts 免费分发，加载速度快
- 对中文回退友好

#### 替代方案

- **Inter** — 更通用的 UI 字体，但缺乏品牌差异化
- **DM Mono** — 适合代码编辑器，但计时器不需要等宽字体

#### 影响

- `global.scss` 顶部 `@import` 引入，`index.html` 添加 preconnect 预连接加速
- 数字使用 `font-variant-numeric: tabular-nums` 确保计时数字不跳动

## 五、问题与解决方案

### 问题1：初始设计系统配色不适合浅色模式

#### 现象

UI-UX-Pro-Max 搜索 "Timer & Pomodoro" 返回的调色板主要是深色方案（Background: `#0F172A`），浅色模式用户也需要合理配色。

#### 根因

搜索结果的 `Timer & Pomodoro` 类型假设用户偏好深色专注模式，未提供对应的浅色方案。

#### 解决方案

额外搜索 `color` 域的 `Productivity Tool` 类型，获得 Teal 系配色：
- Primary: `#0D9488` (Teal)
- Accent: `#EA580C` (Action Orange)
- Background: `#F0FDFA` (Warm Mint White)

将这套配色用于浅色模式 `:root` 变量，深色模式使用 Timer & Pomodoro 的 Focus Red 方案。

#### 最终结果

双模式配色完成，浅色/深色均有完整的品牌识别和 WCAG AA 对比度。

### 问题2：Vue scoped style 无法覆盖 Element Plus 深层组件

#### 现象

`TimerControls.vue` 中 `.el-button--large` 的选择器在 scoped 模式下无法穿透到 Element Plus 组件内部。

#### 根因

Vue 的 `<style scoped>` 会给当前组件的元素添加 `data-v-xxx` 属性，但 Element Plus 的按钮内部元素没有该属性。

#### 解决方案

使用 `:deep()` 穿透选择器：

```css
:deep(.el-button--large) {
  min-width: 150px;
  height: 52px;
}
```

#### 最终结果

按钮样式正确应用，主按钮带有发光阴影效果。

## 八、性能优化记录

### 优化前

- 字体使用系统默认（无外部请求），但缺乏品牌识别
- 过渡动画使用统一 `0.3s ease`
- 阴影使用硬编码 `rgba(0,0,0,0.1)`

### 优化后

- 添加 Google Fonts（Plus Jakarta Sans），通过 preconnect 减少 DNS/TLS 握手延迟
- 过渡动画使用 Bezier 缓动（`cubic-bezier(0.16, 1, 0.3, 1)`），更自然
- CSS 变量统一管理阴影，支持深色模式下增加阴影强度

### 优化措施

- `index.html` 中添加 `<link rel="preconnect">` 到 Google Fonts
- 所有过渡统一使用 `--transition-fast/base/slow` 变量
- 深色模式阴影使用更深的 `rgba(0,0,0,0.2-0.5)` 以补偿低亮度

## 九、技术债务

### 技术债务1：图标使用 Emoji

当前应用使用 Emoji（⏱️☀️🌙🔊🔇✅🎯▶️☕🖥️）作为图标，UI-UX-Pro-Max 设计系统明确建议使用 SVG 图标（Heroicons / Lucide）。

后续优化方案：引入 `@element-plus/icons-vue` 或 `lucide-vue-next` 替换所有 emoji 图标。

### 技术债务2：Plus Jakarta Sans 中文字体回退

Plus Jakarta Sans 不含中文字符，当前回退链为 `'Noto Sans SC', sans-serif`。Google Fonts 完整中文字体加载较慢。

后续优化方案：评估使用中文本地字体优先（如 `PingFang SC`），或在构建时将 CJK 字体子集嵌入。

## 十、开发复盘

### 本次最大收获

- **UI-UX-Pro-Max 设计工具的价值**：通过搜索工具快速获得完整的配色方案、字体推荐和风格指南，避免了主观猜测。特别是产品类型与配色方案的 1:1 匹配非常精准。
- **CSS Variables 的威力**：统一设计变量后，28 个文件的样式修改变得一致且高效。未来调整主题只需改一处。
- **Exaggerated Minimalism 风格对计时器应用非常适配**：计时类应用的交互极为单一，视觉上做减法反而让产品更有质感。

### 本次踩过的坑

- UI-UX-Pro-Max 的 `--design-system` 命令默认推荐 Pattern（落地页结构），对于非营销类的工具型应用，需要手动筛选 color/style/typography 子搜索结果。
- 深色模式下的阴影需要比浅色模式更浓（`opacity` 更高），因为暗背景下微弱阴影不可见。初始用了和浅色模式相同的阴影透明度导致卡片没有立体感。

### 如果重新设计

- 可以先跑一次 `--persist` 命令将设计系统持久化到 `design-system/MASTER.md`，后续所有页面都参照同一份设计源，减少手动协调成本
- 应该先建立 Component Token（按钮、输入框、表格等）的统一规范，再逐个组件实施，这样不同组件的样式更一致
