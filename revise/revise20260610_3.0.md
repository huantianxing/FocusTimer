## 四、打包运行时错误修复

### 错误1：`ModuleNotFoundError: No module named 'webview'`

- **根因**：PyInstaller 用 base 环境打包，该环境无 pywebview
- **修复**：改用 FTimer conda 环境（`conda run -n FTimer`）

### 错误2：`UnicodeEncodeError: 'gbk' codec can't encode '❌'`

- **根因**：Windows 控制台 GBK 编码不支持 emoji
- **修复**：`sys.stdout.reconfigure(encoding='utf-8', errors='replace')` + 所有输出用纯 ASCII

### 错误3：`DLL load failed while importing _ssl`

- **根因**：conda 的 OpenSSL DLL（libssl-3-x64.dll、libcrypto-3-x64.dll）未被 PyInstaller 自动发现
- **修复**：`--add-binary` 显式添加两个 DLL 文件

## 五、PyWebView 桌面窗口布局修复

### 问题：开始计时按钮不可见

#### 现象

PyWebView 窗口（1200x800）中，计时字体巨大（120px），整个页面内容被推到底部，按钮在可视区域之外，用户无法点击。

#### 根因

`font-size: clamp(40px, 10vw, 72px)` — 在 1200px 宽的窗口中，10vw = 120px，是预期的两倍大。加上大量 padding/gap，计时面板占据 ~400px，概览卡片 + 图表又占 ~500px，总内容超过 800px 窗口高度。

#### 解决方案

| 属性 | 之前 | 之后 | 效果 |
|------|------|------|------|
| 字体 | `clamp(40px, 10vw, 72px)` | `clamp(36px, 6vw, 72px)` | 1200px 宽时 = 72px |
| TimerDisplay padding | `32px 24px` | `16px 24px` | 省 16px |
| 卡片 padding | `48px 32px 32px` | `24px 32px` | 省 24px |
| 主内容区 padding | `32px` | `24px` | 省 8px |
| 卡片间距 | `24px` | `16px` | 省 8px |
| 按钮高度 | `52px` | `48px` | 省 4px |

总计节省约 **60px 垂直空间**，确保按钮在 800px 窗口高度内可见。

## 六、一键打包脚本（build_exe.bat）

```

  ├── [1/3] npm run build     → 构建前端

  └── [3/3] 验证输出大小
```

以后修改任何代码后，双击此脚本即可重新打包。

## 八、技术收获

1. **PyInstaller + conda**：必须用包含所有依赖的 conda 环境打包，不能依赖 base
2. **Windows 编码陷阱**：print 中不能用 emoji，需 `sys.stdout.reconfigure(encoding='utf-8')`
3. **OpenSSL DLL**：conda 的 SSL 库在 `Library/bin/`，PyInstaller 不会自动扫描，需 `--add-binary`
4. **PyWebView 视口计算**：`clamp()` 的 vw 单位在桌面窗口中和浏览器中不同，字体要保守
